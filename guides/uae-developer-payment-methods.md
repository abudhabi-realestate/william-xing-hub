# 阿联酋开发商支持的付款方式

> 期房 / 现房 · 银行汇款 · Escrow · 分期结构 · 现金 / 加密货币  
> 基于阿联酋 RERA / ADREC、Escrow 法规及常见开发商 SPA 实务整理。  
> **仅供参考，不构成法律、税务或投资建议。具体以 SPA、开发商通知及持牌机构要求为准。**

---

## 先说一句底线

在阿联酋买开发商房产，**正规路径只有一条：按 SPA 付款计划，把款项付至开发商认可的收款渠道，并最终进入 Escrow 监管账户。**

市场上常见误区包括：「现金可以直接给销售个人」「打私账更快」「施工期也能按揭」。这些说法里，**多数不符合监管要求，部分直接构成资金安全风险。**

💬 **区分：** 「全款现金 / 加密货币付给开发商正规财务渠道」与「交给销售私收」——前者在阿联酋很普遍；后者仍是红线。

签 SPA 前，至少确认三件事：

1. **收款渠道是不是官方授权？**（Escrow 账户、开发商 Finance 或书面指定的 crypto 收款方式）
2. **付款节点和比例是不是写进 SPA？**（口头承诺不算）
3. **付款凭证能不能完整保留？**（Receipt、链上记录、银行回单——金签 / 按揭 / 转售都要用）

|----------|------------------|-------------|----------|----------|
| **银行转账（AED / SWIFT）** | ✅ 主流 | ✅ 主流 | ★★★★★ | 进 Escrow；保留 SWIFT / 水单 |

| **第三方 / 亲属代付** | ⚠️ 视项目 | ⚠️ 视项目 | ★★★ | 须证明资金来源；SPA 买方姓名与付款人关系要可解释 |

## 一、银行转账

### 怎么付

- **境内转账**：买家阿联酋本地账户 → 开发商 **Escrow Account**（AED）
- **境外 SWIFT**：按 SPA / Payment Request 汇至 Escrow（币种多为 AED 或 USD）
- **每次付款**：对照 SPA 节点 + 开发商 **Payment Request / Invoice**（写明单元号、期次、金额）

### 必须核对的信息

| 核对项 | 说明 |
|--------|------|
| **账户名称** | 须为项目 **Escrow Account**，不是开发商母公司或销售个人 |
| **账号 / IBAN** | 与 SPA 附件或 RERA/ADREC 公示信息一致 |
| **Reference / 备注** | 写清 Unit No.、Buyer Name、Payment Milestone |
| **金额** | 与当期节点一致；多付少付都可能影响登记或交房 |

### 凭证要留什么

- 银行 **SWIFT 确认 / 转账回单**（PDF）
- 开发商 **Receipt / Acknowledgement**
- Payment Request 邮件或系统截图  
→ **黄金签证**（累计付款 ≥ 200 万 AED）、**按揭申请**、**转售** 时都会用到。

## 二、Escrow 监管：钱为什么不能打私账

### 迪拜

期房销售受 **RERA** 监管，房款须进入 **RERA 批准的 Escrow Account**。施工进度与放款挂钩，降低「收了钱不建」的风险。

### 阿布扎比

**Law No. 2 of 2025** 进一步强化了 Escrow 与买家保护要求。Day 11 流程里强调：施工期所有分期进 **Escrow**，不是开发商私账。

👉 **解读：** 任何绕开 Escrow 的付款，都在牺牲法律保护和日后办证能力。

## 三、付款计划（Payment Plan）：结构怎么读

期房的核心不是「一次性付清」，而是 **SPA 里的 Payment Schedule**。常见结构：

| 结构类型 | 示意 | 现金流特点 | 常见场景 |
|----------|------|------------|----------|
| **前重后轻** | 5% 预订 + 55% 施工期 + 40% 交房 | 施工期压力大 | 传统阿联酋期房 |
| **均匀分期** | 5% + 每季 5%—8% + 20%—30% 交房 | 较平滑 | 近年部分 launch 为吸引买家 |
| **低首付 launch** | 5%—10% 预订 + 其余拉长 | 入门低，总周期更长 | 热销盘预售 |
| **交房大头** | 10%—20% 施工期 + 60%—70% 交房 | 前期轻，交房前压力集中 | 需提前规划尾款 / 按揭 |

### 施工期分期的触发方式

- 按 **工程里程碑**：地基、结构、封顶、内装等（与现场进度挂钩）
- 按 **时间**：每 3 / 6 个月固定比例（须写进 SPA）
- 收到 **Payment Request** 后通常在 **14—30 天** 内付款（具体看 SPA grace period）

### 三个实务要点

1. **延迟付款有罚息** — 看清 penalty rate 和宽限期  
2. **施工期一般不能按揭** — 银行通常在 **交房尾款** 才介入  
3. **付款计划不是可谈判的口头优惠** — 以 SPA 白纸黑字为准；销售承诺的「特殊 plan」未写进合同 = 无效

## 四、支票、现金与其他「线下」方式

### Manager's Cheque / Cashier's Order

- 阿联酋本地仍常见，尤其 **交房尾款** 或习惯走柜台的买家
- **抬头必须开 Escrow 账户名**，不是开给个人
- 交付后仍建议索取开发商 **正式 Receipt**，并与 SPA 节点对应

### 现金

**目前大部分开发商支持全款现金支付**——尤其在 **现房**、或买家希望 **一次性付清 / 按 SPA 节点用现金结清** 时。

常见正规路径：

| 路径 | 说明 |
|------|------|
| **开发商财务柜台** | 在 Sales Center / Finance 部门缴现，当场入账项目 **Escrow**，开具 **Official Receipt** |
| **银行柜台存现** | 现金存入指定账户或换取 **Manager's Cheque**（抬头 Escrow）后交付 |
| **按节点现金付** | 施工期每一 milestone 均可现金结清，前提是开发商接受且款项进 Escrow |

实务要点：

- **可以现金，但不能给销售个人** — 必须经开发商 **授权收款点 / 银行渠道** 进入 Escrow  
- **AML / KYC** — 大额现金会要求 **护照、来源说明**；单笔或累计超一定金额，银行 / 开发商可能进一步问询  
- **凭证同样关键** — Receipt、存款单与 SPA 节点一一对应；**金签、转售** 仍看付款证明链  

💬 阿布市场 **87% 为 cash buyer**（Day 3，指不按揭、自有资金），其中 **本地在境买家用现金全款** 很常见。

### 信用卡

- 部分开发商接受 **预订金 / 小比例签约款**（如 5% 以内）
- 大额房款因 **手续费、拒付风险、监管** 几乎不会走信用卡
- 若销售说「全程刷卡」——需高度警惕

## 五、按揭（Mortgage）：什么时候才能用

| 阶段 | 能否按揭 | 说明 |
|------|----------|------|
| **预订 / 签约** | ❌ | 自付预订金 + 首付 |
| **施工期分期** | ❌ 一般不行 | 银行为 **完工资产** 放贷；施工期靠自有资金 |
| **交房尾款** | ✅ | 银行评估后放款至 Escrow 或指定账户；买家付 down payment 差额 |
| **现房 / 二手** | ✅ | 标准按揭流程；down payment 通常 20%—25% 起（非居民视银行政策） |

💬 **非居民买家**：各银行对 **LVR、收入证明、国籍** 要求不同，宜在 **签约前** 做 pre-approval，避免交房时尾款缺口。

## 六、交房后分期（Post-handover Payment Plan）

部分 **迪拜** 开发商或合作机构提供 **Handover 后 2—5 年分期**，例如 60% 交房付清、40% 交后分 36 个月。

**注意区分：**

- 这是 **交房后的融资安排**，不是施工期 Escrow 分期  
- 可能有 **额外利息 / 服务费**  
- 条款在 **单独 Schedule 或 MoU**，与施工期 Payment Plan 不是一回事  
- **阿布扎比** 主流 Aldar 等项目以 **施工期 Escrow 分期** 为主，交房后分期不如迪拜普遍

## 七、加密货币（USDT / BTC 等）

**目前大部分开发商支持加密货币支付房款**——以 **USDT** 最为常见，部分项目也接受 **BTC、ETH** 等，具体看开发商政策。

### 常见操作方式

| 步骤 | 说明 |
|------|------|
| **确认收款信息** | 向开发商 **Finance 部门** 索取官方钱包地址或合作支付链接——**不要用销售个人提供的地址** |
| **按 AED 等值支付** | 通常按当日约定汇率折算 USDT 等值；金额、单元号、付款节点写在 Payment Request 里 |
| **完成链上转账** | 保留 **Tx Hash / 截图**；开发商确认到账后开具 **Receipt**，款项计入 Escrow 或项目监管体系 |
| **按 SPA 节点付** | 与银行转账、现金一样，可按 **预订金、施工 milestone、尾款** 分次用 crypto 支付 |

### 实务要点

- **支持 ≠ 可以私下交易** — 须走开发商 **书面授权的渠道**，与现金同理  
- **凭证双份留** — 链上记录 + 开发商 Official Receipt，缺一不可  
- **汇率与时点** — 以开发商确认到账时的 **书面汇率为准**，避免口头报价  
- **各项目政策不同** — 接受的币种、是否收 gas / 手续费、最低金额，签 SPA 前问清 Finance  

💬 迪拜 crypto 产业成熟，部分开发商已与 **持牌支付机构** 合作；阿布主流开发商近年也陆续开放 USDT 等选项——**以具体 launch 的 Payment Method 清单为准**。

## 八、阿布扎比 vs 迪拜：付款实务差异

| 维度 | 阿布扎比 | 迪拜 |
|------|----------|------|
| **监管** | ADREC + DARI；Escrow 强化（2025 法） | RERA + Oqood / DLD |
| **登记费** | 房价 **2% ADREC**（多在签约前后） | 4% DLD + 行政费等（结构不同） |
| **期房分期** | 施工期 Escrow 分期为主 | 同上，另见部分 post-handover 产品 |
| **默认买家结构** | **87% 现金**（Day 3）— 分期靠自有资金 | 按揭比例相对更高 |
| **Crypto** | 主流开发商逐步开放 | 接受面更广，部分项目标配 USDT |
| **凭证用途** | 金签 200 万 AED 累计付款证明 | 金签 + 各银行按揭 |

## 九、付款常见问题（FAQ）

**Q1：预订金必须付吗？**

多数 launch 要求 **5%—10% 预订金** 锁单元，随后签 SPA。预订金通常计入首付，但 **refundable 条件** 看 booking form——未签 SPA 前务必看清。

**Q2：可以加快付款换折扣吗？**

少数促销允许 **early settlement** 折扣，须 **书面 addendum**；不要口头约定。

**Q3：施工期能暂停付款吗？**

不能单方面暂停。进度不达标可依据 SPA **争议 / 投诉** 途径处理，不是随意停付。

**Q4：developer 延迟交房，我还要付尾款吗？**

看 SPA **delay / force majeure** 条款；严重延迟可能有 compensation 或解约机制——签前读清。

**Q5：付款凭证和 DARI 登记有什么关系？**

DARI 登记认的是 **SPA + 规费**；付款凭证是 **履约证明**。金签、按揭、转售时，凭证链条比「登记本身」更常被追问。

**Q6：远程买家怎么付？**

线上面签 + **银行转账 / 加密货币（若项目支持）** 为主；可授权 POA，授权范围建议覆盖 **付款确认、收 Receipt**。

**Q7：SC / 服务费和房款怎么付？**

**SC（Service Charge）** 是交房后交给物业 / 社区，**不走 Escrow 房款账户**；别和房款 milestone 混淆。

**Q8：用 USDT 付，Receipt 上写 AED 还是 USDT？**

通常 Receipt 按 **AED 等值** 入账项目账务；链上 Tx Hash 作为附件——以开发商 Finance 出具为准。

## 十、签 SPA 前付款相关检查清单

| # | 检查项 |
|---|--------|
| 1 | SPA 附件 **Payment Schedule** 每一期比例、日期 / 里程碑 |
| 2 | **Escrow 账户** 全名、IBAN、开户行与官方公示一致 |
| 3 | 延迟付款 **grace period + penalty** |
| 4 | 交房尾款是否预留 **按揭 pre-approval** 空间 |
| 5 | 登记费 **2%（阿布）/ DLD（迪拜）** 谁承担、何时付 |
| 6 | 是否接受 **第三方代付 / crypto** 及所需材料 |
| 7 | 全部凭证 **归档方式**（云盘 + 纸质备份 + 链上记录） |
